
НАЙДЕННЫЕ ДЕФЕКТЫ В КЛАССЕ StringUtils

1. Дефект в методе capitalize()
   - Описание: При передаче строки из одного пробела " " метод возвращает " " вместо ""
   - Ожидаемое поведение: Строка из пробелов должна считаться пустой
   - Воспроизведение: utils.capitalize(" ") возвращает " " вместо ""

2. Дефект в методе to_list()
   - Описание: При передаче строки с пустыми элементами "a,,b" возвращает ["a", "", "b"]
   - Ожидаемое поведение: Пустые элементы должны фильтроваться или обрабатываться особым образом
   - Воспроизведение: utils.to_list("a,,b") возвращает список с пустой строкой

3. Дефект в методе is_empty()
   - Описание: При передаче None возвращает True
   - Ожидаемое поведение: None не является строкой, должно вызываться исключение или возвращаться False
   - Воспроизведение: utils.is_empty(None) возвращает True вместо ожидаемого исключения

4. Дефект в методе list_to_string()
   - Описание: При передаче списка с пустыми строками ["a", "", "c"] возвращает "a, , c"
   - Ожидаемое поведение: Пустые строки должны обрабатываться явно
   - Воспроизведение: utils.list_to_string(["a", "", "c"]) возвращает строку с лишними пробелами

ПРИОРИТЕТЫ ИСПРАВЛЕНИЯ:
- Высокий: Дефекты 1 и 3 (критическая логика)
- Средний: Дефекты 2 и 4 (побочные эффекты)
